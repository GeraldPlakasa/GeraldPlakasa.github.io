Router KPI Check

Project information

  • Category: Automation
  • Project date: February 2023
  • Project URL: -

     The portfolio showcases a comprehensive implementation of Python programming to establish SSH connections with tens of thousands of routers from five different vendors: Huawei, Ericsson, Cisco, ZTE, and Juniper. The purpose of these SSH connections is to meet Key Performance Indicators (KPIs) in the network infrastructure. The initial command executed involves retrieving the CRC (Cyclic Redundancy Check) or the count of lost packets from each router. Subsequently, the application executes the RSl command, retrieving Receiver Power, Transmitter Power, and the router's status.

     This portfolio entry emphasizes the significant undertaking of leveraging Python programming to establish SSH connections with a large number of routers across multiple vendors. The ability to seamlessly communicate with routers from different manufacturers highlights the versatility and efficiency of Python as a programming language.

     The implementation primarily aims to fulfill KPI requirements by collecting essential network data. The initial command retrieves the CRC count, which serves as a metric for packet loss. Additionally, the application executes the RSl command to extract crucial information such as Receiver Power, Transmitter Power, and the router's status. These parameters provide valuable insights into the performance and health of the network infrastructure.
