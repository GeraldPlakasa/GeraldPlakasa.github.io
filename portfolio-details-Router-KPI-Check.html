The Challenge

Manual Monitoring Tidak Scalable

Tim network operations harus memantau performa puluhan ribu router setiap hari. Dengan 5 vendor berbeda (Huawei, Ericsson, Cisco, ZTE, Juniper), setiap router punya command syntax yang berbeda.

Pain Point: Pengecekan manual memakan waktu 8+ jam per hari dan sering terjadi human error dalam pembacaan metrics.

KPI yang harus dipantau termasuk CRC errors (packet loss), Receiver Power, Transmitter Power, dan status konektivitas—semuanya kritikal untuk SLA pelanggan.

The Solution

Unified Multi-Vendor Automation

Saya membangun sistem Python yang mampu establish SSH connection ke semua vendor dengan satu interface. Sistem secara otomatis mendeteksi vendor dan menjalankan command yang sesuai.

  • Auto-detection: Identifikasi vendor otomatis berdasarkan banner dan response pattern
  • Parallel Processing: Concurrent connections untuk speed up 10x dari sequential
  • Error Handling: Retry mechanism dengan exponential backoff untuk koneksi gagal
  • Structured Output: Data di-normalize ke format unified untuk analisis
The Impact

Dari 8 Jam Menjadi Menit

Tool ini langsung diadopsi sebagai bagian dari daily operations team. Penghematan waktu yang signifikan memungkinkan team fokus ke tasks yang lebih strategic.
